    PandasAI Integration for Optopsy Chat UI¶

    Motivation¶

    The Optopsy chat UI (optopsy/ui/) is built on Chainlit + LiteLLM with a custom tool-calling agent loop. Users load options chain CSVs and run backtests. Currently, data exploration is limited to pre-built tools (preview_data, describe_data, create_chart), which means every possible data question must be anticipated and coded as a tool.

    We investigated PandasAI as a way to handle arbitrary free-form data exploration queries (statistics, filtering, aggregation, ad-hoc visualization), letting us focus our custom tooling on what's truly domain-specific: options backtesting.

    What is PandasAI?¶

    PandasAI v3 (released Oct 2025, MIT license) translates natural language into Python/pandas code, executes it, and returns typed results. It supports multiple LLM backends via a LiteLLM integration package.

    import pandasai as pai
from pandasai import Agent
from pandasai_litellm import LiteLLM

pai.config.set({"llm": LiteLLM(model="anthropic/claude-haiku-4-5-20251001")})

df = pai.read_csv("options_data.csv")
response = df.chat("What is the average bid-ask spread by expiration?")

    How It Achieves Accuracy¶

    1. Schema context — sends column names, dtypes, and sample rows (df.head()) to the LLM so it understands the data structure
    2. Code generation — LLM produces Python code that assigns a result to a structured output dict
    3. Error correction — if generated code fails, the error is fed back to the LLM for retry (configurable max_retries, default 3)
    4. Result type detection — responses are typed via isinstance() checks:
    Response Type Value Use Case
    NumberResponse int/float "What's the average delta?"
    StringResponse str "What symbol is this?"
    DataFrameResponse pd.DataFrame "Show top 10 strikes by volume"
    ChartResponse base64 image / file path "Plot bid-ask spread over time"

    Key Capabilities¶

    • Accepts existing pandas DataFrames directly (no need for pai.read_csv())
    • Multi-DataFrame queries via Agent([df1, df2])
    • Conversation memory with agent.follow_up()
    • Synchronous API — works in thread pool executors
    • LiteLLM integration supports any model (Claude, GPT, Gemini, local via Ollama)
    • Built-in retry logic with error self-correction

    Limitations¶

    • Python <3.12 only — does not support Python 3.12 or 3.13
    • Matplotlib only for charts — no Plotly support; charts returned as base64 images
    • No built-in sandboxing — uses raw exec() with no restricted builtins
    • Known RCE vulnerability (CVE-2024-12366) — prompt injection can bypass code filters
    • Custom skills require enterprise license — @pai.skill() decorator is enterprise-only
    • No streaming — synchronous only, no token streaming

    Python Compatibility Analysis¶

    Our Constraint¶

    pyproject.toml specifies requires-python = ">=3.12,<3.14". Our dev environment runs Python 3.13.

    Can We Downgrade?¶

    We performed an exhaustive search of the entire codebase (optopsy/ and tests/) for Python 3.12+ features:

    Feature Status
    PEP 695: type statement Not used
    PEP 695: Generic syntax (class Foo[T]:, def foo[T]()) Not used
    PEP 698: @override decorator Not used
    PEP 701: Nested f-strings Not used
    itertools.batched() Not used
    pathlib.Path.walk() Not used
    typing.Buffer Not used
    Match statement type guards Not used

    Result: The codebase contains zero Python 3.12+ features. The >=3.12 constraint was inherited from the original pandas-ta dependency (since replaced with pandas-ta-classic, which supports 3.9-3.13).

    All dependencies also support 3.11: - pandas-ta-classic — Python >=3.9 - empyrical-reloaded — Python >=3.9 (recommends 3.10+) - pandas, numpy, pydantic, chainlit, litellm — all support 3.11

    Conclusion: Downgrading to Python 3.11 is safe.

    Current Chat UI Architecture¶

    What Could Be Offloaded to PandasAI¶

    These tools handle generic data exploration that PandasAI can do natively:

    Tool Lines What It Does
    preview_data ~50 Show head/tail of dataset
    describe_data ~200 Column stats, nulls, distributions
    create_chart (dataset source) ~300 Ad-hoc Plotly charts from raw data

    Plus any arbitrary pandas question users might ask (which currently gets no answer or a generic LLM text response).

    Estimated savings: ~500-600 lines of data inspection/charting code could be simplified or removed over time.

    What Must Stay as Custom Tooling¶

    Tool Why PandasAI Can't Handle It
    run_strategy / scan_strategies Calls optopsy library with domain-specific strike selection, leg pairing, Greeks filtering
    build_signal / signal composition TA indicator computation + signal-date-to-option-date alignment
    simulate Monte Carlo portfolio simulation with options-specific P&L accounting
    compare_results / query_results Results interpretation with DTE/delta bucketing
    check_data_quality Strategy-specific validation (e.g., "enough strikes for iron condor?")
    suggest_strategy_params Domain knowledge about DTE/delta distributions
    load_csv_data Column remapping + optopsy normalization

    Integration Design¶

    Tool: ask_data¶

    A new tool registered in the existing tool handler system that delegates to PandasAI.

    Parameters: - question (str, required) — natural language question about the dataset - dataset_name (str, optional) — which dataset to query; defaults to active dataset

    Flow: 

    User: "What is the average bid-ask spread?"
        │
        ▼
OptopsyAgent (existing LiteLLM loop)
  → selects ask_data tool
        │
        ▼
_handle_ask_data()
  1. Resolve dataset via _require_dataset()
  2. Configure PandasAI with OPTOPSY_MODEL
  3. Create Agent([df])
  4. response = agent.chat(question)
  5. Format response → ToolResult
        │
        ▼
ToolResult(llm_summary, user_display, chart_figure=None, result_df=...)

    Response formatting:

    PandasAI Type llm_summary user_display Extra
    NumberResponse str(value) str(value) —
    StringResponse value value —
    DataFrameResponse Shape + head(10) Full markdown table result_df for CSV download
    ChartResponse "Chart generated" ![chart](data:image/png;base64,...) Inline image

    Chart handling: PandasAI produces matplotlib charts as base64 images. These are embedded as inline markdown images in user_display. Chainlit renders markdown images natively. Users who need interactive Plotly charts can still use the existing create_chart tool.

    Security Considerations¶

    PandasAI executes LLM-generated Python code via exec() with no restricted builtins. This is a known vulnerability (CVE-2024-12366).

    Mitigations: - Our chat UI is a local/self-hosted tool, not a public-facing API - PandasAI only has access to the DataFrame passed to it - No filesystem/network access is needed for data analysis - Future: could wrap PandasAI's execution in a Docker sandbox (PandasAI provides a Sandbox base class)

    Alternative Considered: Build Our Own¶

    We also designed a lightweight (~200 line) alternative that replicates PandasAI's core approach: - LLM generates pandas code via litellm.completion() - Code executed in sandbox with restricted builtins (reusing pattern from _signals_builder.py) - Generates Plotly charts natively (vs PandasAI's matplotlib) - Error retry with error message feedback - No external dependency

    Trade-offs:

    Aspect PandasAI Build Our Own
    Python version 3.11 only Any (3.11+)
    Code to maintain ~150 lines (wrapper) ~200 lines (full impl)
    Chart library Matplotlib (static images) Plotly (interactive)
    Security No sandbox (raw exec) Restricted builtins sandbox
    Accuracy features Built-in retry, code validation, memory Manual retry, manual prompting
    Dependencies +2 packages (pandasai, pandasai-litellm) None
    Conversation memory Built-in follow_up() Not needed (outer agent handles it)

    Decision: Use PandasAI. The built-in retry logic, code validation, and battle-tested prompting provide better accuracy out of the box. The matplotlib limitation is acceptable since our existing create_chart tool handles interactive Plotly charts for complex visualizations.
